# 🚗 자동차 판매량 데이터 수집 프로그램

사용자가 시작 연도와 끝 연도를 설정하면 해당 연도 범위의 모든 월별 자동차 판매량 데이터를 자동으로 수집하여 SQLite 데이터베이스에 저장하는 프로그램입니다.

## 📋 기능

- **연도 범위 설정**: 시작 연도와 끝 연도를 직접 입력하여 원하는 연도 범위 설정
- **전체 월별 수집**: 설정된 연도 범위의 모든 월별 데이터를 자동으로 크롤링
- **데이터베이스 저장**: SQLite를 사용하여 수집된 데이터 저장
- **진행 상황 표시**: 각 연도/월별 수집 진행 상황을 실시간으로 표시
- **데이터 조회**: 수집 완료 후 저장된 데이터를 순위별로 조회

## 📦 설치

### 1. 필요한 패키지 설치
```bash
pip install -r requirements.txt
```

### 2. 데이터베이스 초기 설정
```bash
python database_setup.py
```

## 🚀 사용법

### 메인 프로그램 실행
```bash
python car_sales_crawler.py
```

### 입력 순서
1. **시작 연도 입력**: 수집 시작 연도 입력 (예: 2022)
2. **끝 연도 입력**: 수집 끝 연도 입력 (예: 2025)
3. **자동 수집**: 설정된 연도 범위의 모든 월별 데이터를 자동으로 수집
4. **데이터 조회**: 수집 완료 후 저장된 데이터 조회 여부 선택

## ⚠️ 주의사항

- **미래 데이터 방지**: 현재 연도 이후의 데이터는 경고 후 사용자 확인을 받습니다.
- **서버 부하 방지**: 월별 수집 간 2초의 대기 시간을 두어 네이버 서버에 부하를 주지 않습니다.
- **데이터 유효성**: 네이버 검색 결과 페이지의 구조가 변경될 경우 크롤링이 실패할 수 있습니다.
- **사용 목적**: 수집된 데이터는 교육 및 개인적인 용도로만 사용하세요.
- **수집 시간**: 연도 범위가 클수록 수집 시간이 오래 걸릴 수 있습니다.

## 🔧 문제 해결

### 크롤링 실패 시
1. 네이버 검색 결과 페이지가 정상적으로 로드되는지 확인
2. 인터넷 연결 상태 확인
3. 페이지 구조 변경 여부 확인
4. 입력한 연도에 데이터가 실제로 존재하는지 확인

### 데이터베이스 오류 시
1. `database_setup.py`를 다시 실행하여 데이터베이스 재생성
2. 파일 권한 확인

### 입력 오류 시
- 연도: 2000년~2030년 사이의 숫자만 입력
- 끝 연도는 시작 연도보다 크거나 같아야 함
- 현재 연도 이후 입력 시 경고 메시지 표시
